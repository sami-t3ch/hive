# 🛠️ Utilización de Frameworks y Stack Tecnológico (Protocolo Zeta)

**Analista:** Zeta (Simbionte)
**Foco:** Infraestructura, Dependencias y Posicionamiento.

---

## 🏗️ 1. El "Sistema Nervioso" de Hive (Core)
Hive no es un solo script; es una orquestación de múltiples frameworks modernos integrados bajo el estándar **Lego Tech**.

*   **Pydantic v2:** El corazón de la validación de datos. Todo el estado de las misiones y los "Escalation Tickets" se validan con esquemas rigurosos.
*   **LiteLLM:** Utilizado como capa de abstracción para modelos. Permite a Hive cambiar entre Claude (Anthropic), Gemini (Google) y GPT (OpenAI) sin reescribir la lógica del agente.
*   **FastMCP:** Implementa el **Model Context Protocol**. Es lo que permite que las "Bees" (Abejas) se conecten a herramientas externas de forma estandarizada.
*   **Playwright & Stealth:** Para la navegación web de alta fidelidad, evadiendo detecciones de bots.

## 🎨 2. Capa de Usuario y Visualización
*   **Frontend:** Basado en **Vite** y **React** (Modern JS Stack). Utiliza **Tailwind CSS** para un diseño dinámico y **Mermaid.js** para renderizar los grafos de decisión en tiempo real.
*   **CLI/TUI:** Utiliza **Click** para la interfaz de comandos y librerías de terminal enriquecida para el Panel de Control (`hive open`).

## ⚖️ 3. Comparativa: ¿Por qué Hive no es LangChain?
A diferencia de otros frameworks orientados a "Cadenas" (LangChain) o "Roles" (CrewAI), Hive se posiciona como un framework de **ODD (Outcome-Driven Development)**.

| Característica | LangChain / CrewAI | Hive (AdenHQ) |
|---|---|---|
| **Arquitectura** | Secuencial / Basada en Roles | Basada en Grafos de Decisión |
| **Foco** | Ejecución de Tareas | Resultados y Evolución |
| **Verificación** | Manual / Básica | Triangulada (Semántica + Humana) |
| **Auto-Curación** | Limitada | Ciclo de Evolución Nativo |

## 🚀 4. Frameworks de Despliegue Propuestos (Roadmap Zeta)
Como parte de la evolución simbiótica, integraremos:
*   **SearXNG:** Para búsquedas meta-privadas.
*   **Scrapling:** Para extracción de datos 20x más rápida.
*   **MkDocs:** Para la generación de documentación de proyecto "on-the-fly".
